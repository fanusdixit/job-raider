# `results.json` schema (Phase 1)

Generated by Job Raider after each successful pipeline run (`job_raider.pipeline.run` ‚Üí merge ‚Üí `job_raider.storage.write_results_atomic`).

**Canonical reference:** `_bmad-output/planning-artifacts/architecture-job-raider-phase1.md` ¬ß8‚Äì10.

---

## Top level

| Field | Type | Description |
|-------|------|-------------|
| `schema_version` | int | Must be **`1`**. Other values cause `ResultsLoadError` on load. |
| `generated_at` | string | UTC wall time when the file was written, ISO 8601 with **`Z`** suffix. |
| `tool_version` | string | Job Raider version string (e.g. from `job_raider.__version__`). |
| `searches` | array | One block per **configured** search, ordered by **`search_id` ascending** (empty searches still appear). |
| `source_runs` | array | Per-source outcomes from the **latest** pipeline run (for the HTML run report). Omitted in older files; new writes always include the key (possibly `[]`). |

---

## `source_runs[]` (optional on read; always written on new runs)

| Field | Type |
|-------|------|
| `search_id` | string |
| `search_name` | string |
| `source_label` | string ‚Äî source `label` from YAML |
| `status` | string ‚Äî `"ok"` or `"error"` |
| `item_count` | int ‚Äî opportunities after normalize + keyword filter for that source this run |
| `error_detail` | string or **`null`** |

---

## `searches[]`

| Field | Type |
|-------|------|
| `id` | string ‚Äî search id from `searches.yaml` |
| `name` | string ‚Äî display name from config |
| `items` | array of opportunity objects (may be empty) |

---

## `items[]`

| Field | Type |
|-------|------|
| `dedupe_id` | string ‚Äî stable merge key (see **Dedupe id** below) |
| `title` | string |
| `source` | string ‚Äî source `label` from YAML |
| `url` | string ‚Äî absolute `http`/`https` URL |
| `published_at` | string (ISO 8601) or **`null`** |
| `last_seen_at` | string ‚Äî UTC with **`Z`**; see semantics below |
| `search_id` | string |
| `search_name` | string |

---

## `dedupe_id` normalization

`dedupe_id` is derived from the opportunity URL by `job_raider.dedupe.compute_dedupe_id` (architecture ¬ß8.1):

1. Parse with `urllib.parse.urlparse`.
2. Scheme must be **`http`** or **`https`**; host must be present.
3. Scheme and host are lowercased.
4. **Fragment** (`#...`) is stripped.
5. **Default ports** are omitted from the serialized netloc (`:80` on http, `:443` on https).
6. Userinfo, path, params, and query are preserved (Phase 1 does **not** strip tracking query parameters).

**Known limitation:** two distinct listings that share the same normalized URL collapse to one row.

---

## `published_at` and `null`

- **`null`:** No publication time was parsed from the feed/HTML. The dashboard **does not** show the ‚ÄúNew‚Äù badge for that row (see architecture ¬ß9).
- **String:** ISO 8601 timestamp; may include offset. When loading from disk, naive values are interpreted as **UTC** for storage compatibility (the HTML renderer may treat naive `published_at` differently when computing the badge‚Äîsee `job_raider.generate_dashboard`).

---

## `last_seen_at` semantics

- Updated to the **current run‚Äôs** `generated_at` (UTC) **only** when the opportunity appears in this run‚Äôs set **after** fetch + keyword filter.
- If a source fails or an item stops matching keywords, `last_seen_at` is **not** refreshed.
- **Edge case:** an item can disappear from the dashboard after **30 days** without being seen, even if it still exists on the remote site (operator fixes the source, widens keywords, or deletes `results.json` to cold-start).

---

## Retention (prune)

Before each write, records where **`now_utc - last_seen_at > 30 days`** are removed. Records **on** the 30-day boundary are **kept** (`<= 30 days` retained).

---

## Sort order of `items`

Within each search block, `items` is sorted deterministically:

1. Rows with non-null `published_at`: **descending** by publication time.
2. Rows with **`null` `published_at`:** after all dated rows, sorted by **`title`** ascending (**casefold**).
3. Tie-breaker for same timestamp: **`title`** ascending (casefold).

---

## Malformed or incompatible file

On load, invalid JSON or schema violations raise **`ResultsLoadError`** with a path-specific message (e.g. `invalid JSON`, `unsupported schema_version`, bad field types).

**Recovery:** fix the JSON, or **delete** `results.json` to cold-start an empty store (you will lose historical merge state).

## Minimal example

```json
{
  "schema_version": 1,
  "generated_at": "2026-03-19T14:00:00Z",
  "tool_version": "0.1.0",
  "searches": [
    {
      "id": "example_search",
      "name": "Human readable title",
      "items": [
        {
          "dedupe_id": "https://example.com/job/1",
          "title": "Example role",
          "source": "Example RSS",
          "url": "https://example.com/job/1",
          "published_at": "2026-03-18T10:00:00+01:00",
          "last_seen_at": "2026-03-19T14:00:00Z",
          "search_id": "example_search",
          "search_name": "Human readable title"
        },
        {
          "dedupe_id": "https://example.com/job/2",
          "title": "Undated listing",
          "source": "Example portal",
          "url": "https://example.com/job/2",
          "published_at": null,
          "last_seen_at": "2026-03-19T14:00:00Z",
          "search_id": "example_search",
          "search_name": "Human readable title"
        }
      ]
    }
  ]
}
```
