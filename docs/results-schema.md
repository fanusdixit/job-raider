# `results.json` schema (Phase 1)

Generated by Job Raider after each successful merge (see `job_raider.merge`, `job_raider.storage`).

## Top level

| Field | Type | Description |
|-------|------|-------------|
| `schema_version` | int | Must be `1`. |
| `generated_at` | string | UTC timestamp, ISO 8601 with `Z`. |
| `tool_version` | string | Job Raider package version. |
| `searches` | array | One block per configured search (`search_id` ascending). |

## `searches[]`

| Field | Type |
|-------|------|
| `id` | string — search id from `searches.yaml` |
| `name` | string — display name from config |
| `items` | array of opportunity objects (may be empty) |

## `items[]`

| Field | Type |
|-------|------|
| `dedupe_id` | string — normalized canonical URL (merge key) |
| `title` | string |
| `source` | string — source label from YAML |
| `url` | string — canonical http(s) URL |
| `published_at` | string (ISO 8601) or `null` |
| `last_seen_at` | string (UTC `Z`) — updated when the item appears in a run’s fetch set |
| `search_id` | string |
| `search_name` | string |

## Semantics

- **`last_seen_at`**: refreshed only when the opportunity is present in the **current run** after fetch + keyword filter. If a source fails, items missing from that run do not refresh and may be **pruned** after **30 days** without appearing (see PRD).
- **Retention**: records with `now_utc - last_seen_at > 30 days` are removed on write.
- **Sort order** in each `items` array: non-null `published_at` descending; then items with null `published_at`, sorted by `title` (case-insensitive ascending); dated ties broken by `title` casefold ascending.

## Malformed file

If JSON is invalid or required fields are missing, `load_results_state` raises `ResultsLoadError`. Delete or fix `results.json`, or remove it to cold-start an empty store.
